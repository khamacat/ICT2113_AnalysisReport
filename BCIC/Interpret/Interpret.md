## Revised Requirements (Codified)

### BO - 1.1 Registration and Staff Onboarding
- **BO - 1.1.1 Admin Account**  
  The system shall provide an administrative account for the Admin to manage staff access to the portal.
- **BO - 1.1.2 Log-in**  
  Staff shall be able to access academic administration functions according to their role.  
  The system shall allow staff to log in to the portal using authenticated credentials.
- **BO - 1.1.3 Customise Profile**  
  The system shall allow staff accounts to customise their profile.
- **BO - 1.1.4 Account Management**  
  The system shall allow the Admin to view all staff accounts and profiles registered.

### BO - 1.2 Course Administration
- **BO - 1.2.1 Course & Subject Management**  
  The system shall allow Admin to set up and organise all courses and their related subjects.
- **BO - 1.2.2 Student Registration & Enrollment**  
  The system shall allow the Admin to register students and assign them to the appropriate courses and subjects.
- **BO - 1.2.3 Register Staff**  
  The system shall allow staff to be assigned to appropriate courses and subjects.

### BO - 1.3 Timetable Scheduling
- **BO - 1.3.1 Create Slot**  
  The system shall allow the Admin to create timetable slots by specifying course, subject, date/day, start time, end time (or duration), and location.
- **BO - 1.3.2 Automatic Scheduling**  
  The system shall generate staff and time-slot suggestions based on defined constraints and availability.
- **BO - 1.3.3 Manually Assign Staff to Slots**  
  The system shall allow the Admin to vet and, if conflicts arise, manually overwrite staff assignments to an available slot.
- **BO - 1.3.4 Approve Timetable**  
  The system shall allow the Admin to approve and publish the final timetable to the respective staff.

### BO - 1.4 Attendance Management
- **BO - 1.4.1 Record Attendance**  
  The system shall capture and record student attendance automatically using a QR code during lessons, with a fallback option to record manually.
- **BO - 1.4.2 Track Student Attendance History**

### BO - 1.5 Monitoring & Reporting
- **BO - 1.5.1 Attendance Summaries**  
  The system shall retrieve and review attendance information for administrative and academic purposes.
- **BO - 1.5.2 Generate Attendance Reports**  
  The staff shall be able to summarise attendance reports when requested.
